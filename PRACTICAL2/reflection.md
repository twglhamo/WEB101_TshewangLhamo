# Reflection on Weather API Dashboard

##  Documentation

In this practical, I worked with REST APIs and implemented different HTTP methods such as GET, POST, PUT, and DELETE.

- I used the **GET method** to fetch weather data from the OpenWeatherMap API.
- The **POST method** was used to save location data to a mock API (JSONPlaceholder).
- I implemented **PUT** to update existing saved locations.
- I used **DELETE** to remove saved locations.

I also structured the application using tabs to separate different functionalities, which made the UI easier to understand and use.

## What I Learned

Through this practical, I learned:
- How APIs work and how to fetch data using JavaScript
- How to send data using POST requests
- How to update and delete data using PUT and DELETE methods
- How to handle API responses and display them on the webpage
- How frontend and backend communication works

This helped me understand how real-world web applications function.


## Challenges Faced

One of the main challenges I faced was:
- Connecting the buttons properly with JavaScript functions
- Handling API errors and making sure the data displays correctly
- Fixing issues in the edit (PUT) functionality

At first, some buttons were not working because of incorrect IDs in the HTML. I fixed this by carefully matching the IDs in both HTML and JavaScript.

Another issue was with the edit modal. The input fields were not updating correctly, so I had to correct the IDs and structure of the modal.



##  How I Overcame Challenges

I solved these problems by:
- Checking the browser console for errors
- Comparing HTML and JavaScript IDs
- Testing each feature step by step
- Debugging and correcting small mistakes


##  Conclusion

This practical helped me gain a better understanding of REST APIs and how to build interactive web applications. It also improved my debugging skills and attention to detail.

Overall, this was a very useful and practical learning experience.
